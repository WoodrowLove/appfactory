# Packaging & Shipping

## Overview

Once an app passes quality gates and monetization is configured, the Shipper agent handles everything needed to get it on the App Store: icon generation, screenshot capture, promo video creation, listing optimization, and binary upload.

## App Icon Generation

### Tool: Nano Banana Pro (Google AI)

**Process:**
1. Shipper generates a prompt based on the app's one-pager (category, color scheme, core concept)
2. Calls Nano Banana Pro API to generate a 1024x1024 icon
3. Validates the icon meets Apple's requirements (no transparency, no rounded corners — Apple applies these)
4. Runs a differentiation check against all other AppFactory icons

**Prompt template:**
```
Design a minimal, modern iOS app icon for [app_name], a [category] app that [one-sentence description].

Style: Clean, flat design with a single recognizable symbol.
Color palette: [primary color] with [secondary accent].
Must NOT include: text, words, letters, photographs, or complex scenes.
Must include: One clear, instantly recognizable symbol that represents [core concept].
Background: Solid or subtle gradient in [color range].
Resolution: 1024x1024 pixels.
```

**Icon differentiation rules:**
- No two AppFactory apps may share the same primary color
- No two icons may use the same symbol
- Icons should be recognizable at 60x60px (home screen size)
- Style should vary: some geometric, some organic, some abstract

### Fallback: If Nano Banana Pro produces a subpar icon
The Shipper requests 3 variants, picks the best based on:
1. Simplicity at small sizes
2. Color distinctiveness
3. Relevance to the app concept

If all 3 are poor, flag for human review.

## Screenshot Generation

### Tool: Fastlane Snapshot

**Process:**
1. Shipper writes XCUITest scripts that navigate to each key screen
2. Fastlane runs these tests on multiple simulators simultaneously
3. Screenshots are captured at exact moments defined in the test scripts
4. Fastlane frameit adds device bezels and marketing text

### XCUITest Script Template

```swift
import XCTest

class Screenshots: XCTestCase {
    let app = XCUIApplication()

    override func setUpWithError() throws {
        continueAfterFailure = false
        app.launchArguments = ["--screenshot-mode"]  // Skip onboarding, load demo data
        app.launch()
    }

    func testScreenshot01_Home() {
        // Navigate to home screen
        // Wait for content to load
        snapshot("01_Home")
    }

    func testScreenshot02_MainFeature() {
        // Navigate to the primary feature screen
        // Interact to show it in an interesting state
        snapshot("02_MainFeature")
    }

    func testScreenshot03_Detail() {
        // Navigate to a detail view with sample data
        snapshot("03_Detail")
    }

    func testScreenshot04_Premium() {
        // Show a premium feature in action
        snapshot("04_Premium")
    }

    func testScreenshot05_Settings() {
        // Show settings or another key screen
        snapshot("05_Settings")
    }
}
```

### Screenshot Mode

The app must support a `--screenshot-mode` launch argument that:
- Skips onboarding
- Loads demo/sample data
- Disables animations (for clean captures)
- Sets a consistent date/time (for reproducibility)
- Hides the status bar clock or sets it to 9:41 (Apple's standard)

The Builder includes this mode in every app.

### Device Matrix

| Device | Screen Size | Required By Apple |
|--------|-------------|-------------------|
| iPhone 16 Pro Max | 6.9" | Required (largest iPhone) |
| iPhone 16 | 6.1" | Recommended |

## Promo Video

### Tool: Remotion

Remotion generates programmatic videos using React. The Shipper creates a video composition that:

1. Opens with the problem statement (text overlay, 3 seconds)
2. Shows the app solving the problem (screen recording, 15 seconds)
3. Highlights 2-3 key features (animated text + screen captures, 9 seconds)
4. Closes with the app name and "Download Now" CTA (3 seconds)

**Total: 30 seconds** (Apple's maximum for app preview videos)

### Video Specifications (Apple Requirements)
- Duration: 15-30 seconds
- Resolution: Match device (1290x2796 for iPhone 16 Pro Max)
- Frame rate: 30fps
- Format: H.264, .mp4
- Audio: Optional (many users browse with sound off)
- Content: Must show the actual app running (no live-action, no external footage)

### Screen Recording for Video

The Shipper uses `xcrun simctl io` to record the simulator:

```bash
# Start recording
xcrun simctl io booted recordVideo --codec h264 output.mp4

# Stop recording (Ctrl+C or after timeout)
```

The Remotion composition then edits, trims, and adds text overlays to the raw recording.

## App Store Listing (ASO)

### ASO — App Store Optimization

The listing is the app's sales page. Every word matters for both search ranking and conversion.

### Title (30 characters max)

Format: `[App Name] - [Primary Keyword]`

Example: `RoutineRest - Sleep Habits`

Rules:
- App name must be unique (not trademarked by another company)
- Primary keyword should match the highest-volume search term from research
- No keyword stuffing

### Subtitle (30 characters max)

One compelling benefit statement.

Example: `Build your perfect bedtime routine`

Rules:
- Complements the title (doesn't repeat it)
- Focuses on benefit, not feature
- Uses a high-value keyword not in the title

### Description (4000 characters max)

Structure:
```
[Opening hook — 1-2 sentences stating the problem]

[Value proposition — how this app solves it]

KEY FEATURES:
• [Feature 1] — [benefit]
• [Feature 2] — [benefit]
• [Feature 3] — [benefit]
• [Feature 4] — [benefit]
• [Feature 5] — [benefit]

PREMIUM FEATURES:
• [Premium 1] — [benefit]
• [Premium 2] — [benefit]
• [Premium 3] — [benefit]

[Social proof or credibility statement]

[Free trial CTA]

PRICING:
• Free: [what's included]
• Premium: $X.XX/month or $XX.XX/year
• 7-day free trial included

[Privacy statement — we take your privacy seriously]

[Support info — email, website]
```

### Keywords (100 characters max)

Comma-separated, no spaces after commas.

Rules:
- Don't repeat words from the title or subtitle (Apple already indexes those)
- Use singular, not plural (Apple matches both)
- Don't use competitor names (Apple may reject)
- Prioritize: specificity > volume (long-tail keywords convert better)
- Use all 100 characters

### What's New

For v1.0, keep it simple:
```
Welcome to [App Name]!

We're excited to launch with:
• [Feature 1]
• [Feature 2]
• [Feature 3]

We'd love to hear your feedback — reach out at [support email].
```

## TestFlight Soak Test

Before submitting to App Store Review, every build is pushed to TestFlight for a 24-hour soak test. This catches runtime crashes, memory leaks, and edge-case failures that compilation and static analysis miss.

**Process:**
1. Shipper uploads the build to TestFlight via `fastlane pilot upload`
2. A 24-hour timer starts in `state.json` (`soak_test_started_at`)
3. During the soak period, the build runs on available test devices
4. After 24 hours, the Shipper checks for crash reports via App Store Connect API
5. If zero crashes: proceed to App Store submission
6. If crashes detected: route back to Builder with crash symbolication data

**Configuration in `openclaw.yaml`:**
```yaml
shipping:
  testflight_soak_test:
    enabled: false          # Disabled by default until TestFlight is set up
    duration_hours: 24      # Minimum soak time before submission
    max_crash_threshold: 0  # Any crash fails the soak test
    auto_submit_after: true # Automatically submit to review if soak passes
```

Enable this once your Apple Developer account has TestFlight configured with at least one test device or internal tester group. The soak test adds 24 hours to the pipeline but significantly reduces the risk of a crash-related rejection (Guideline 2.1).

## Phased Release (Canary Deployment)

Instead of a binary ship-or-don't-ship decision, use Fastlane's `phased_release` option to roll out updates gradually over 7 days:

**Rollout Schedule:**
| Day | Percentage | Cumulative Users |
|-----|-----------|-----------------|
| 1 | 1% | 1% |
| 2 | 2% | 2% |
| 3 | 5% | 5% |
| 4 | 10% | 10% |
| 5 | 20% | 20% |
| 6 | 50% | 50% |
| 7 | 100% | 100% |

**Implementation:**
```ruby
# In Fastfile, update the upload lane:
lane :upload do
  deliver(
    submit_for_review: false,
    automatic_release: false,
    phased_release: true,   # Enable phased rollout
    force: true,
    metadata_path: "./metadata",
    screenshots_path: "./screenshots"
  )
end
```

**Monitoring during rollout:**
- The Shipper checks App Store Connect ratings daily during the phased release
- If the app's average rating drops below **3.5 stars**, the Shipper pauses the rollout via the App Store Connect API
- A paused rollout triggers a notification to the Router, which flags the project for human review
- If the rating recovers (after a hotfix release), the rollout can be resumed
- If the rating does not recover within 48 hours of pausing, consider pulling the release entirely

Phased releases are especially valuable after the first few apps are live, as they protect the developer account's overall health by catching user-facing issues before full exposure.

## Xcode Cloud Alternative

Apple provides **25 free compute hours per month** with Xcode Cloud, included with every Apple Developer Program membership. This can offload builds from your local Mac and enable parallel builds.

**Benefits:**
- Frees up the local Mac for other tasks while builds run in the cloud
- Enables parallel builds across multiple apps simultaneously
- Apple-managed infrastructure means no Xcode version or simulator management
- Built-in TestFlight distribution (builds go straight to TestFlight after success)
- Integrates with App Store Connect for seamless submission

**Trade-offs:**
- 25 hours/month limits throughput to roughly 12-15 full builds (each ~1.5-2 hours including tests)
- Less control over the build environment compared to local Fastlane
- Requires source code to be hosted on a supported git provider (GitHub, GitLab, Bitbucket)
- Additional compute hours beyond 25 cost $14.99/month per 25-hour block

**Recommendation:** Consider Xcode Cloud as a complement to local Fastlane builds rather than a full replacement. Use it when the local Mac is busy with another build, or when you need to run builds for multiple apps concurrently. The free tier is sufficient for early-stage operations (2-3 apps/month), and the cost scales reasonably as throughput increases.

## Submission Pipeline

### Fastlane Configuration

```ruby
# Fastfile
default_platform(:ios)

platform :ios do
  desc "Build the app"
  lane :build do
    match(type: "appstore")  # Fetch signing certificates
    build_app(
      scheme: ENV["APP_SCHEME"],
      output_directory: "./build",
      export_method: "app-store"
    )
  end

  desc "Take screenshots"
  lane :screenshots do
    snapshot(
      scheme: ENV["APP_SCHEME"] + "UITests",
      devices: [
        "iPhone 16 Pro Max",
        "iPhone 16",
        "iPad Pro 13-inch (M4)"
      ],
      output_directory: "./screenshots",
      clear_previous_screenshots: true
    )
    frameit(silver: false)
  end

  desc "Upload to App Store Connect"
  lane :upload do
    deliver(
      submit_for_review: false,  # Human approval required
      automatic_release: false,
      force: true,
      metadata_path: "./metadata",
      screenshots_path: "./screenshots",
      skip_binary_upload: false,
      precheck_include_in_app_purchases: true
    )
  end

  desc "Full pipeline: build + screenshots + upload"
  lane :ship do
    build
    screenshots
    upload
  end
end
```

### Code Signing

Managed by `fastlane match`:
- Certificates and profiles stored in a private git repo (encrypted)
- `match(type: "appstore")` fetches the correct signing identity
- One signing identity for all AppFactory apps (same developer account)

### Pre-submission Checklist

The Shipper verifies before uploading:

- [ ] Icon is 1024x1024, no transparency, no rounded corners
- [ ] Screenshots exist for all required device sizes
- [ ] App preview video is 15-30 seconds, correct resolution
- [ ] Title, subtitle, description are within character limits
- [ ] Keywords use all 100 characters
- [ ] Privacy URL is live and accessible
- [ ] Support URL is live and accessible
- [ ] Age rating is set correctly
- [ ] Content rights declarations are accurate
- [ ] Export compliance is marked (usually "No" for non-crypto apps)
- [ ] IDFA usage is declared correctly
- [ ] Privacy manifest matches the app's actual data collection

### Submission Queue

Apple limits 5 apps in review per developer account. The factory manages this:

```json
// In factory.db
{
  "submission_queue": [
    {"slug": "routine-rest", "ready_since": "2026-03-01T10:00:00Z", "priority": 1},
    {"slug": "focus-timer", "ready_since": "2026-03-02T14:00:00Z", "priority": 2}
  ],
  "in_review": [
    {"slug": "budget-buddy", "submitted": "2026-02-28T09:00:00Z"},
    {"slug": "meal-planner", "submitted": "2026-02-28T11:00:00Z"}
  ],
  "max_in_review": 5
}
```

When a review completes (approve or reject), the Shipper notifies the Router, which promotes the next queued app.

## Post-Submission

### If Approved
1. State transitions to `approved`
2. Shipper sets release date (immediately or scheduled)
3. Marketer is notified to begin launch content
4. State transitions to `live`

### If Rejected
1. Shipper reads the rejection reason from App Store Connect
2. Categorizes the rejection:
   - **Metadata issue**: Fix listing, resubmit (no code change)
   - **Code issue**: Send back to Builder with rejection details
   - **Policy issue**: May require spec changes (send to Architect)
3. Route to appropriate agent via Router
4. Increment rejection counter in `state.json`
5. After 3 rejections, flag for human review

### Common Rejection Reasons (and how we prevent them)

| Rejection Reason | Prevention |
|-----------------|------------|
| 2.1 Performance: App Completeness | Quality Gate 3 (Feature Completeness) |
| 2.3 Performance: Accurate Metadata | Quality Gate 5 (App Store Compliance) |
| 4.3 Design: Spam | Differentiation check in Gate 5 + Research validation |
| 5.1.1 Privacy: Data Collection | Privacy manifest verification in Gate 2 |
| 5.1.2 Privacy: Data Use and Sharing | AI data disclosure check in Gate 5 |
| 3.1.1 Business: Payments | StoreKit 2 template ensures compliance |
| 3.1.2 Business: Subscriptions | Clear terms in paywall template |
